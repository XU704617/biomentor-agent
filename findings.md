# BioMentor Agent 研究发现

## 2026-05-26 项目交接文档阅读

### 项目概述
- **项目名称**: BioMentor Agent | 智造学伴
- **项目类型**: AI 教育平台 / 生物制造课程智能体 / 自适应学习系统 / 科研训练平台
- **一句话定位**: 面向生物制造课程的科研型自适应学习智能体平台，通过领域 RAG 知识库、多智能体测评诊断、科研案例辅导和教师学习分析看板，实现从知识学习到科研能力训练的闭环培养。

### 技术栈
- **前端**: Next.js + TypeScript + Tailwind CSS + shadcn/ui + Recharts/ECharts
- **后端**: FastAPI + Python + SQLAlchemy + SQLite (demo) / PostgreSQL (prod)
- **AI/RAG**: PyMuPDF + python-docx + Chroma + Embedding API + LLM API
- **Agent**: LangGraph 风格状态机 + JSON Schema + structured outputs

### 五层架构（来自调研报告）
1. **评价层**: RAG 可信度、题目质量、评分一致性、RAGAS + 人工样例集 + 日志分析
2. **应用层**: 教师端、学生端、报告、看板，React/Next.js + FastAPI
3. **智能体层**: 出题、评分、诊断、推荐、科研辅导，LangGraph 风格状态机 + JSON Schema
4. **知识层**: chunk、embedding、metadata、知识点，Chroma/Qdrant + 关系库
5. **数据层**: 教材、课件、论文、案例、答题记录，对象存储 + SQLite/PostgreSQL

### 推荐实现路线（5个阶段）

**阶段 A：真实可用底座**
- 文件上传、解析、chunk、embedding、向量检索、课程/资料/知识点基本数据模型
- 对应单元: 01, 02, 20

**阶段 B：测评闭环**
- AI 出题、题目质控、教师审核、测验发布、学生答题、客观题评分
- 对应单元: 04, 05, 06

**阶段 C：诊断与报告**
- 主观题 rubric 评分、知识点诊断、能力画像、学习报告、错题本
- 对应单元: 07, 08, 09, 10, 11

**阶段 D：冲奖增强**
- 科研案例库、案例辅导 Agent、教师端班级看板、教学建议、知识地图可视化
- 对应单元: 03, 12, 13, 14, 16, 17

**阶段 E：评估与包装**
- 人工评估集、演示数据、可复现实验、PPT 和讲解脚本
- 对应单元: 21, 22, 25

### 竞品借鉴矩阵

| 竞品 | 借鉴什么 | 不能照搬什么 | 对应单元 |
|------|---------|------------|---------|
| NotebookLM | 以用户资料为知识边界、引用来源、资料摘要 | 偏通用阅读助手，不负责课程测评和班级诊断 | 1/2/14 |
| Khanmigo | Socratic tutor、提示式辅导 | 内容体系不是生物制造，不强调科研案例 | 10/13/24 |
| Moodle/Canvas | 课程-题库-测验-成绩簿信息架构 | 传统LMS智能化不足 | 6/15/16 |
| Gradescope | rubric、分项评分、教师复核 | 不是RAG知识库和自适应学习平台 | 5/7 |
| Quizlet/Duolingo | 短任务、即时反馈、强化练习 | 不适合专业课程科研能力训练 | 11/24 |
| Elicit/SciSpace | 论文摘要、文献问答、证据引用 | 需转化为课程教学任务 | 12/14 |

### 评估指标体系

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 资料解析成功率 | ≥ 90% | 文本型 PDF/Word/TXT 范围 |
| 题目教师可接受率 | ≥ 70% | 人工审核通过或少量修改后通过 |
| AI 评分与教师相关性 | ≥ 0.75 | 小样本演示即可 |
| 教师备题时间节省 | ≥ 50% | 对比手工生成10道题 |

### 第一版禁止事项
- 不做复杂登录权限系统
- 不做移动 App
- 不做重型 Neo4j 知识图谱
- 不做完整学习通平替
- 不做无审核直接发布 AI 题目
- 不做完全自由的多智能体聊天系统

### 工程原则（来自调研报告）
1. 所有 AI 输出必须结构化 JSON
2. 所有 RAG 输出带 source_refs
3. 每个 workflow 有 validate_node 做格式和证据校验
4. 失败自动 retry with critique
5. 先让系统有形，再让系统变聪明
6. 所有重要动作必须入日志
7. 所有报告必须能追溯到题目、资料和证据片段
