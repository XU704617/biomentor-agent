# 05_DEVELOPMENT_TASKS.md

# 开发任务拆解

## Stage 1：项目骨架 + 静态 UI

### 目标

先让系统有形，跑通页面和 mock 演示流程。

### 任务

- 创建 `frontend/` Next.js 项目
- 创建 `backend/` FastAPI 项目
- 配置基础 README 和 `.env.example`
- 完成教师端页面框架
- 完成学生端页面框架
- 使用 mock 数据跑通页面跳转

### 验收

- 前端可以本地启动
- 后端可以本地启动
- 教师端和学生端主要页面可访问
- 页面视觉风格统一

---

## Stage 2：资料上传与解析

### 任务

- 实现文件上传 API
- 支持 PDF/DOCX/TXT
- 保存文件到 uploads
- 写入数据库
- 实现文本解析
- 实现文档分块
- 前端资料库页面接入真实 API

### 验收

- 教师可以上传文件
- 系统显示解析状态
- 能看到文本摘要和 chunk 数量

---

## Stage 3：RAG 知识库

### 任务

- 接入 embedding 服务
- 接入 Chroma
- 将 chunk 写入向量库
- 实现 `/api/rag/search`
- 实现 `/api/rag/answer`

### 验收

- 可以根据问题检索相关资料片段
- 返回结果带来源 file/page/chunk

---

## Stage 4：AI 自动出题

### 任务

- 设计出题 JSON schema
- 编写出题 prompt
- 实现 `/api/questions/generate`
- 将生成题目存入数据库
- 前端 AI 出题页面接入 API

### 验收

- 可以选择知识点生成题目
- 题目包含题干、选项、答案、解析、来源
- 教师可以查看生成结果

---

## Stage 5：题库审核与测验发布

### 任务

- 实现题目编辑
- 实现批准/驳回
- 实现测验创建
- 实现题目组卷
- 实现测验发布

### 验收

- 教师可以审核题目
- 教师可以发布测验

---

## Stage 6：学生答题与评分

### 任务

- 实现学生测验页面
- 实现 attempt/response 数据模型
- 实现客观题自动评分
- 实现主观题 AI 评分初版

### 验收

- 学生可以完成测验
- 系统可以自动给出分数和解析

---

## Stage 7：学习报告与教师看板

### 任务

- 实现学生报告 API
- 实现能力画像计算
- 实现报告页面
- 实现教师班级分析 API
- 实现教师分析看板

### 验收

- 学生能看到报告
- 教师能看到班级分析
- 有能力雷达图和知识点错误率

---

## Stage 8：科研案例辅导

### 任务

- 预置 5 个生物制造案例
- 实现案例辅导页面
- 实现引导式 AI 反馈
- 实现实验设计评分

### 验收

- 学生可以进入案例
- AI 能追问并给出反馈

---

## Stage 9：竞赛展示增强

### 任务

- 增强首页视觉
- 增加五层架构图
- 增加演示模式
- 准备 demo 数据
- 优化整体 UI 质感

### 验收

- 可以完整演示项目闭环
- 页面有明显高级科技感和生物制造主题
