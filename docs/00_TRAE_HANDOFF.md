# BioMentor Agent｜智造学伴 Trae Solo 交接说明

## 你正在接手什么项目

你正在接手一个新项目：**BioMentor Agent｜智造学伴**。

这是一个面向生物制造课程的科研型自适应学习智能体平台，目标不是做普通刷题系统，而是做一个融合：

- 生物制造领域知识库
- AI 自动出题
- 智能批改
- 学习诊断
- 个性化学习路径
- 科研案例辅导
- 教师教学决策看板

的真实可演示 MVP。

---

## 项目一句话定位

> 面向生物制造课程的科研型自适应学习智能体平台，通过领域 RAG 知识库、多智能体测评诊断、科研案例辅导和教师学习分析看板，实现从知识学习到科研能力训练的闭环培养。

---

## 第一阶段目标

当前不是做完整商业系统，而是先完成一个**可运行、可展示、可继续扩展**的 MVP。

第一阶段必须跑通这个闭环：

```text
教师上传资料
→ 系统解析资料
→ 构建课程知识库
→ AI 自动生成题目
→ 教师审核题目
→ 发布测验
→ 学生答题
→ 系统自动评分
→ 生成学习诊断报告
→ 教师查看班级分析
```

---

## 推荐技术栈

### 前端

```text
Next.js
TypeScript
Tailwind CSS
shadcn/ui
Recharts 或 ECharts
```

### 后端

```text
FastAPI
Python
SQLAlchemy
SQLite 起步，后续可迁移 PostgreSQL
```

### AI / RAG

```text
PyMuPDF：PDF 解析
python-docx：Word 解析
Chroma：向量数据库
Embedding API：文本向量化
LLM API：出题、批改、诊断、报告生成
```

---

## 第一优先级

先完成：

1. 项目目录和前后端骨架
2. 教师端静态页面
3. 学生端静态页面
4. Mock 数据演示闭环
5. 文件上传与资料解析
6. RAG 检索
7. AI 自动出题
8. 学生答题与报告

---

## 禁止事项

第一版不要做：

- 复杂登录权限系统
- 移动 App
- 多租户 SaaS
- 复杂考试监控
- 完整学习通平替
- 重型 Neo4j 知识图谱
- 自动联网抓全网文献
- 完全自由的多智能体聊天系统
- 无审核直接发布 AI 题目

---

## 工程原则

1. 先让系统有形，再让系统变聪明。
2. 先做可演示闭环，再做工程增强。
3. 所有 AI 输出必须结构化 JSON。
4. 所有题目、解析、报告必须可追溯到资料来源。
5. 第一版可以 mock 用户，但不要只做纯静态壳。
6. 页面风格要高级、科技感、生物制造主题、适合竞赛展示。
7. 每完成一个阶段，都要能本地运行演示。
