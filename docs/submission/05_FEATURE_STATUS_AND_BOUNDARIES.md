# 05_FEATURE_STATUS_AND_BOUNDARIES.md — 功能状态与能力边界

## 说明

本文档对 BioMentor Agent 各核心模块的当前实现状态进行如实描述，明确区分"已实现能力"和"仍需完善"的部分，为申报材料的撰写和演示规划提供参考依据。

**状态标识**：

| 标识 | 含义 |
|------|------|
| ✅ 已实现 | 功能完整可演示，前后端联通，核心链路可闭环 |
| 🟡 部分实现 | 基础功能可用，部分高级特性待完善，或前后端未完全打通 |
| 🔵 规划中 | 已有设计方案，尚未进入开发阶段 |

## 全局能力边界说明

### 已实现的全局能力

- **多模块联动机制**：知识图谱、生物工具箱、产业案例、科研实战的结果均可"带入"学术研讨模块，形成模块间数据流转。
- **LLM 双路径策略**：所有 LLM 调用均实现"优先调用真实 API + 失败回退本地兜底"的双路径保障。
- **BFF 容错层**：前端 Next.js API Routes 统一做请求转发、超时控制（30s）和错误响应，避免前端直接暴露后端内部错误。
- **防重复请求守卫**：科研实战、知识图谱 AI 对话等高频调用场景已添加 useRef 防重复请求守卫。

### 当前明确能力边界（不能做的事）

| 能力边界 | 说明 |
|----------|------|
| ❌ 不是完整自动科研系统 | 科研实战模块提供训练任务建议和方法指导，不替代湿实验，不做真实实验数据的自动分析 |
| ❌ 没有真实教学成效数据 | 平台尚未在真实课堂中部署，所有"效果"描述为预期应用场景和演示验证结果 |
| ❌ 外部文献检索未集成 | 知识探索和科研实战的文献推荐基于知识库和 LLM 内部知识，非实时数据库检索 |
| ❌ 不支持语音交互 | 学术研讨模块仅支持文字答辩，无语音识别/合成功能 |
| ❌ 不支持复杂登录权限 | 当前版本为单用户演示模式，无教师/学生角色鉴权和数据隔离 |
| ❌ 不做学习通/MOOC 平替 | 不追求完整的课程管理系统（CMS）功能，聚焦知识探索和科研训练 |
| ❌ 不依赖特定 LLM 厂商 | 但 API 可用性取决于所配置的 LLM 服务，平台本身不提供 LLM 服务 |
