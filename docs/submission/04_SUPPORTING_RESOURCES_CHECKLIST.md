# 02_DEVELOPMENT_AND_APPLICATION_REPORT_DRAFT.md — 开发与应用报告草稿

## 一、开发背景

生物制造是合成生物学与工业生物技术交叉的前沿学科，涵盖分子生物学、细胞工程、代谢工程、发酵调控等广泛领域。当前该领域的教学面临几方面挑战：

**知识体系庞杂且分散**：学生需要同时掌握从基因克隆到工程菌株构建、从代谢通路分析到发酵工艺优化等多层次知识，但传统课程教学以章节递进为主，知识点之间的交叉关联难以直观呈现。

**产业案例教学难度大**：生物制造案例涉及基因回路设计、菌株筛选、工艺放大等工程化背景，学生缺乏将理论知识映射到真实产业场景的结构化工具。

**科研能力训练缺乏系统性**：从知识学习到独立开展科研之间存在明显断档。论文阅读、研究问题定义、实验设计、数据分析、学术报告与答辩等科研核心环节缺少循序渐进的训练体系支撑。

**AI 原生教学工具待开发**：以 ChatGPT 为代表的通用大语言模型虽已广泛渗透到教育领域，但面向特定学科（如生物制造）的教学智能体仍较为稀缺，在领域知识深度、教学场景适配性和结构化任务引导等方面均有较大提升空间。

在上述背景下，我们开发了 **BioMentor Agent（智造学伴）**——一个面向生物制造课程的 AI 辅助学习平台。平台以七个核心模块构建从知识获取到科研实战的完整学习闭环，旨在为学生提供结构化的知识探索路径和科研能力训练场景。

## 二、设计与开发

### （一）平台与技术选择

BioMentor Agent 采用前后端分离架构：

- **前端**：Next.js 14 + TypeScript + Tailwind CSS，基于 App Router 架构。选用 Next.js 的主要考量包括 React 生态成熟度、SSR/SSG 带来的首屏性能优势、以及 API Routes 可兼作 BFF（Backend for Frontend）层，在前后端之间做请求转发和超时处理。
- **后端**：FastAPI (Python) + SQLAlchemy + SQLite。FastAPI 的异步特性适合 LLM 调用场景下的并发请求处理；SQLite 轻量零配置，适合演示环境和教学部署，后续可按需迁移至 PostgreSQL。
- **AI/LLM**：采用 OpenAI 兼容接口接入国产大模型（如 DeepSeek），通过统一的 LLM 服务层封装，支持模型替换而无需修改业务代码。后端提供本地兜底策略，在 API 不可用时自动回退到模板化响应，保证核心功能的可用性。
- **RAG 知识库**：基于向量检索的检索增强生成，对课程资料、产业案例、文献摘要等进行分块索引和语义检索，为 AI 问答提供领域语境。
- **可视化**：Canvas 2D 实现首页粒子背景，SVG 实现知识图谱渐进展开，NGL.js 实现蛋白 3D 结构查看。
- **版本管理**：Git + GitHub，采用 feature-branch 工作流，分支命名规范化。

### （二）开发过程

项目开发全程采用 AI 辅助开发模式，主要开发路径如下：

**第一阶段：项目骨架搭建**。从设计文档（PRD、技术架构、页面清单）出发，使用 AI 代码生成工具搭建 Next.js 前端项目和 FastAPI 后端项目骨架。前期重点完成 15 个页面路由、全局样式系统和流光玻璃设计语言，确保视觉风格统一。

**第二阶段：核心模块迭代开发**。围绕七个核心模块分阶段推进：

1. **知识探索与知识图谱**：先实现静态 RAG 检索和知识星图展示，再接入后端 API 实现语义搜索和 AI 问答。知识图谱从 12 学科星图逐步演进为渐进展开式 BioMind Map，支持六维展开（基础知识、科研前沿、产业应用等）。
2. **生物工具箱**：实现蛋白结构 3D 查看器（支持 PDB/UniProt 搜索）、质粒图谱解读（支持 GenBank/FASTA 上传）、序列分析（ORF 预测、酶切位点识别、引物检查）、代谢通路浏览。每个工具逐步集成 AI 解释面板，实现"工具操作 + AI 解读"的联动。
3. **产业案例**：构建生物制造案例库数据模型，预置多领域案例数据，提供结构化案例详情页（技术路线、研究要点、关键参数），并作为后续科研实战模块的任务输入源。
4. **科研实战**：从静态任务展示页逐步升级为 AI 生成科研任务系统，支持"案例驱动"（基于产业案例生成关联科研任务）和"独立选题"两种模式，将科研流程拆解为文献调研、研究假设、实验设计、数据分析等结构化步骤。
5. **学术研讨**：实现模拟学术答辩训练器，支持论文/报告材料导入（PDF/DOCX/PPT/TXT/MD），AI 生成 Defense Brief，多轮追问答辩，最终输出六维评分报告与薄弱点分析。

**第三阶段：AI 接入与前后端联调**。实现前端 Next.js API Routes 作为 BFF 层转发请求至 FastAPI 后端，增加超时控制、错误兜底和加载状态管理。AI 调用层面实现了"LLM 优先 + 本地兜底"双路径，确保在 API 不可用时核心功能仍可演示。

**第四阶段：功能对齐与质量收尾**。进行前后端功能对齐审计，识别未接通接口和 mock 页面，统一前后端数据 Schema。修复状态管理导致的重复请求和无限加载问题，确保演示流程顺畅。

在开发过程中，团队大量使用提示词工程（Prompt Engineering）与 LLM 协作：结构化 prompt 模板用于任务生成、答辩评分、知识解释等核心 AI 功能；JSON Schema 约束确保 AI 输出可解析、可验证；失败重试和本地兜底机制保障系统鲁棒性。

## 三、应用过程与效果

### 预期应用场景

BioMentor Agent 面向以下教学场景设计：

1. **课堂教学辅助**：教师在讲授生物制造核心概念时，可引导学生通过知识图谱直观理解知识点间的关联，通过生物工具箱查看蛋白结构和代谢通路，增强课堂互动性。
2. **自主学习与课前预习**：学生可在课前通过知识探索模块检索相关知识，建立初步概念框架。
3. **产业案例研讨**：以产业案例为切入点，引导学生分析真实工程技术问题，通过科研实战模块将案例分析转化为结构化研究任务。
4. **科研能力入门训练**：面向高年级本科生或研究生入门阶段，通过科研实战模块生成结构化研究任务，帮助学生理解科研流程和方法。
5. **答辩与报告演练**：学生可导入自己的论文或报告草稿，通过学术研讨模块进行模拟答辩练习，获得 AI 生成的结构化反馈。

### 演示验证

当前版本已完成全链路演示验证：

- 前端 30+ 页面构建通过（`npm run build` 零错误），单元测试 33/33 通过。
- 后端 FastAPI 所有核心 API 接口均可成功响应。
- 前端 Next.js API Routes 与后端接口联调通过。
- 全部七个模块在本地环境下可完整体验，形成了从知识探索到学术研讨的演示闭环。

### 阶段性效果

基于开发过程中的内部演示验证，当前系统已实现以下阶段性成果：

- **功能完整性**：七个核心模块均已实现可演示版本，覆盖知识获取、工具操作、案例分析、科研任务和答辩训练全流程。
- **AI 集成度**：各模块均集成 AI 能力，支持语义搜索、智能问答、任务生成、答辩模拟等场景，且具备 API 不可用时的本地兜底保障。
- **代码可复现性**：完整的 Git 开发历史和分支管理记录，可根据 commit 历史追溯功能演进过程，适合作为 AI 辅助开发的教学案例。
- **教学适用性**：内置生物制造领域的预置数据和案例，可直接用于课堂教学场景演示。

> **重要声明**：上述"阶段性效果"仅基于开发过程中的内部演示验证和功能测试。目前平台尚未在真实课堂环境中部署使用，尚无教学成效的量化数据（如学生成绩提升、学习体验满意度等）。后续需在真实教学场景中开展应用验证，收集实际使用数据。

## 四、创新与反思

### 主要创新

1. **从知识到科研的结构化闭环**：区别于传统 LMS 平台侧重"教—考—评"，BioMentor Agent 打造了"知识探索 → 图谱理解 → 工具操作 → 案例分析 → 科研实战 → 学术答辩"的深度学习路径，在知识获取和科研训练之间建立了循序渐进的桥梁。
2. **AI 原生架构设计**：平台从架构层面将 LLM 作为核心能力组件纳入设计，而非简单地在现有教学系统上叠加 AI 问答。AI 贯穿知识解释、任务生成、答辩模拟等多个场景，且具备模型替换的灵活性。
3. **多模态生物信息呈现**：突破纯文本知识传递的局限，整合 3D 蛋白结构、质粒图谱、代谢通路网络等可视化组件，多维度呈现生物制造核心概念。
4. **开放可复现**：全程 AI 辅助开发的工程实践本身具有教学价值——完整的 Git 历史可用于展示 AI 协作开发的最佳实践和迭代过程，启发更多教育技术开发者。

### 反思与当前限制

在开发过程中，我们认识到当前版本存在以下限制，需要在后续迭代中逐步完善：

**应用验证不足**：平台目前仅在开发团队内部进行了演示验证和功能测试，尚未在真实课堂环境中部署使用。缺乏真实教学场景下的用户反馈、使用数据和教学成效量化指标，这是当前最大的局限性。

**外部文献检索待完善**：当前知识探索和科研实战模块依赖预置知识库和 LLM 内在知识，尚未集成实时文献数据库检索能力（如 PubMed、Google Scholar），限制了科研实战任务的文献调研环节的深度和准确性。

**实验设计/数据分析能力边界**：科研实战模块中的实验设计和数据分析步骤目前为 AI 生成的训练任务建议，提供方法论指导和设计思路参考，而非完整的自动科研系统。它不能替代湿实验操作，也不做真实实验数据的自动分析处理。

**学习评估模块深度不足**：相比其他六个模块，学习评估模块（测评、诊断、轨迹）目前功能相对基础，距离完整的自适应学习诊断体系还有较大差距。

**领域覆盖广度**：当前预置的知识图谱和案例库主要覆盖生物制造核心领域（分子生物学、细胞生物学、结构生物学、合成生物学、生物信息学等），如需扩展到更多交叉学科或细化到具体课程，需要持续补充领域数据。

**评估指标体系待建立**：虽然系统功能已基本完备，但尚未建立系统化的评估指标体系来衡量平台在真实教学场景中的效果（如对学生知识掌握度、科研能力提升的量化评估）。

### 后续改进方向

1. 在真实课程中开展应用试点，收集师生使用数据和反馈。
2. 完善外部文献检索集成，增强科研实战模块的文献调研能力。
3. 深化实验设计模块，增加实验方案合理性校验和优化建议。
4. 升级学习评估模块，建立更完整的自适应诊断和学习路径推荐体系。
5. 扩展知识图谱和案例库覆盖范围，增加更多生物制造子领域内容。</think>

> 【旁白】知识图谱帮助学生直观理解知识点之间的层级和交叉关系，AI 导师面板则提供即时的概念解释和追问功能。

**3:25 – 4:10 生物工具箱**

> 【画面】点击导航栏"生物工具箱"，进入 `/tools` 入口页，展示四个工具卡片（蛋白结构、质粒图谱、序列分析、通路图谱）
>
> 【旁白】生物工具箱是平台的实操模块，集成了四个常用的生物信息学工具。
>
> 【操作1】点击"蛋白结构"，展示蛋白搜索（输入 P00790 pepsin A），显示 3D 结构查看器和 AI 结构解释。
>
> 【旁白】蛋白结构查看器支持 PDB/UniProt 搜索，3D 视图可以旋转、缩放，右侧 AI 面板会同步生成结构特征和功能的解释。
>
> 【操作2】切换到"质粒图谱"页面，展示预置质粒图谱的元件点击和高亮效果。
>
> 【旁白】质粒图谱工具支持经典质粒浏览和自定义 GenBank/FASTA 上传，点击质粒元件可查看功能说明。
>
> 【操作3】快速展示序列分析页面（DNA→RNA 转录、翻译、ORF 预测）。
>
> 【操作4】快速展示通路图谱页面（节点点击、上下游高亮）。
>
> 【旁白】四个工具均集成了 AI 解释功能，实现"工具操作 + 智能解读"的联动体验。

**4:10 – 4:45 产业案例**

> 【画面】点击导航栏"产业案例"，进入 `/cases` 页面，展示案例卡片列表
>
> 【旁白】产业案例库收录了生物制造领域的典型工程案例，每个案例包含技术路线、研究要点和关键参数的结构化展示。
>
> 【操作】点击一个案例进入详情页（如"工程菌株产量优化"），展示案例详情（背景、技术路线、关键数据等）。
>
> 【旁白】产业案例不仅提供结构化的学习材料，还可以直接关联到科研实战模块，作为科研任务生成的输入源。

**4:45 – 5:40 科研实战**

> 【画面】点击案例详情页中的"生成科研任务"按钮（或直接导航至 `/research` 页面）
>
> 【旁白】科研实战模块是平台的核心亮点之一。它支持两种模式：案例驱动模式基于产业案例自动生成关联科研任务，独立选题模式允许学生自由输入研究主题。
>
> 【操作】展示任务生成过程（加载动画），生成完成后展示结构化科研任务卡片。
>
> 【旁白】生成的任务会将科研流程拆解为文献调研、研究假设、实验设计、数据分析、结果讨论等结构化步骤，每个步骤包含具体的方法指导和关键问题提示。
>
> 【操作】展开几个步骤卡片，展示其中的详细内容；快速浏览文献推荐区域。
>
> 【旁白】这个功能帮助学生理解科研流程的系统性和方法论，为从被动学习向主动研究过渡提供支架。

**5:40 – 6:30 学术研讨**

> 【画面】点击导航栏"学术研讨"，进入 `/seminar` 页面
>
> 【旁白】最后一个核心模块是学术研讨——一个模拟学术答辩训练器。
>
> 【操作】展示导入材料界面（支持手动粘贴文本、PDF/DOCX/PPT/PPTX/TXT/MD 上传）。
>
> 【旁白】学生可以导入自己的论文草稿或研究计划，系统会基于材料生成 Defense Brief 答辩策略包。
>
> 【操作】展示 Defense Brief 内容（研究要点总结、潜在问题预测、应对策略建议）；确认后进入答辩界面。
>
> 【旁白】答辩支持基础/标准/挑战三种难度和 3/5/8 轮问答。答辩过程中 AI 评委逐步追问，学生逐轮回答，最后生成包含六维评分、薄弱点分析和学习推荐的结构化报告。
>
> 【操作】展示一两轮问答交互，最后展示最终报告的评分维度（内容完整性、逻辑严密性、创新性、表达能力等）。
>
> 【旁白】学术研讨将知识学习、案例分析、科研实战的输出整合为答辩训练场景，形成完整的学习验证闭环。

**6:30 – 7:00 联动演示**

> 【画面】快速展示各模块之间的联动入口。
>
> 【旁白】值得一提的是，各模块之间并非孤立运行。知识图谱节点可以"带入答辩"；产业案例可以关联科研任务；工具箱中的蛋白和通路也可以导入研讨材料。这种模块联动是平台设计的核心特征。
>
> 【操作】快速展示：知识图谱 → 点击"带入答辩" → 跳转至研讨页面；产业案例 → "生成科研任务" → 跳转至科研实战页面。
