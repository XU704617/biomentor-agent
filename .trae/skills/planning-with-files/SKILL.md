# 文件规划系统 (Planning with Files)

用持久化的 Markdown 文件作为你的「磁盘工作记忆」。

## 快速开始

在任何复杂任务之前：

1. **创建 `task_plan.md`** - 阶段、进度、决策追踪
2. **创建 `findings.md`** - 研究结果、发现记录
3. **创建 `progress.md`** - 会话日志、测试结果

规划文件放在项目根目录。

## 文件用途

| 文件 | 用途 | 更新时机 |
|------|------|---------|
| `task_plan.md` | 阶段、进度、决策 | 每个阶段完成后 |
| `findings.md` | 研究、发现 | 任何发现之后 |
| `progress.md` | 会话日志、测试结果 | 整个会话过程中 |

## 核心规则

### 1. 先创建计划
永远不要在没有 `task_plan.md` 的情况下开始复杂任务。

### 2. 任务分解与阶段化
将复杂任务分解为有序阶段。每个阶段有明确的完成标准。

### 3. 决策前先读取
在做重大决策之前，读取计划文件刷新目标。

### 4. 行动后更新
完成任何阶段后：标记状态、记录错误、记下创建/修改的文件。

### 5. 记录所有错误
每个错误都要写入计划文件：

```markdown
## 遇到的错误
| 错误 | 尝试次数 | 解决方案 |
|------|---------|---------|
| FileNotFoundError | 1 | 创建了默认配置 |
| API 超时 | 2 | 添加了重试逻辑 |
```

### 6. 永远不要重复失败
```
if 操作失败:
    下一步操作 != 同样的操作
```
记录尝试过的方法，改变方案。

## 三次失败协议

```
第1次尝试：诊断并修复 → 仔细阅读错误，找到根本原因，针对性修复

第2次尝试：替代方案 → 换一种方法，不同的工具或库

第3次尝试：重新思考 → 质疑假设，搜索解决方案，考虑更新计划

3次失败后：向用户求助 → 说明尝试了什么，分享具体错误，请求指导
```

## task_plan.md 模板

```markdown
# [项目名称] 任务计划

## 目标
[一句话描述目标]

## 阶段

### 阶段 1: [阶段名称] (状态: pending)
- [ ] 任务 1.1
- [ ] 任务 1.2

### 阶段 2: [阶段名称] (状态: pending)
- [ ] 任务 2.1
- [ ] 任务 2.2

## 遇到的错误
| 错误 | 解决方案 |
|------|---------|
```

## findings.md 模板

```markdown
# [项目名称] 研究发现

## [日期] [主题]
- 关键发现 1
- 关键发现 2

## 决策记录
| 决策 | 理由 | 日期 |
|------|------|------|
```

## progress.md 模板

```markdown
# [项目名称] 进度日志

## [日期/会话]
- 完成了阶段 X
- 创建/修改了文件: ...
- 遇到的问题: ...
```

## 何时使用

**使用场景：**
- 多步骤任务（3步以上）
- 研究任务
- 构建/创建项目
- 跨越多次工具调用的任务

**跳过场景：**
- 简单问题
- 单文件编辑
- 快速查询
